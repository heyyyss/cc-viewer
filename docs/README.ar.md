# CC-Viewer

نظام مراقبة طلبات Claude Code، يلتقط ويعرض بشكل مرئي جميع طلبات واستجابات API الخاصة بـ Claude Code في الوقت الفعلي (النص الأصلي، بدون اختصار). يساعد المطورين على مراقبة السياق الخاص بهم لمراجعة المشكلات واستكشافها أثناء عملية Vibe Coding.
يوفر أحدث إصدار من CC-Viewer أيضًا حلول نشر الخادم للبرمجة عبر الويب، بالإضافة إلى أدوات البرمجة على الأجهزة المحمولة. نرحب بتطبيقه في مشاريعكم، وسيتم فتح المزيد من ميزات الإضافات مستقبلاً مع دعم النشر السحابي.

## طريقة الاستخدام

### التثبيت

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### وضع البرمجة

ccv هو بديل مباشر لـ claude، جميع المعاملات تُمرر إلى claude، مع تشغيل Web Viewer في نفس الوقت.

```bash
ccv                    # == claude (الوضع التفاعلي)
ccv -c                 # == claude --continue (متابعة المحادثة الأخيرة)
ccv -r                 # == claude --resume (استئناف المحادثة)
ccv -p "hello"         # == claude --print "hello" (وضع الطباعة)
ccv --d                # == claude --dangerously-skip-permissions (اختصار)
ccv --model opus       # == claude --model opus
```

بعد تشغيل وضع البرمجة، سيتم فتح صفحة الويب تلقائيًا.

يمكنك استخدام claude مباشرة من صفحة الويب، مع إمكانية عرض رسائل الطلبات الكاملة ومراجعة تغييرات الكود.

والأكثر إثارة، يمكنك حتى البرمجة من جهازك المحمول!


### وضع السجل

⚠️ إذا كنت لا تزال معتادًا على استخدام أداة claude الأصلية، أو إضافة VS Code، يرجى استخدام هذا الوضع.

في هذا الوضع، عند تشغيل ```claude``` أو ```claude --dangerously-skip-permissions```

سيتم تشغيل عملية سجل تلقائيًا لتسجيل سجلات الطلبات في ~/.claude/cc-viewer/*yourproject*/date.jsonl

تشغيل وضع السجل:
```bash
ccv -logger
```

عندما لا يمكن طباعة المنفذ المحدد في وحدة التحكم، المنفذ الأول الافتراضي هو 127.0.0.1:7008. إذا كان هناك عدة منافذ، فإنها تتبع بالترتيب، مثل 7009، 7010

يكتشف هذا الأمر تلقائيًا طريقة تثبيت Claude Code المحلية (NPM أو Native Install) ويتكيف معها.

- **إصدار NPM من claude code**: يحقن تلقائيًا سكريبت الاعتراض في ملف `cli.js` الخاص بـ Claude Code.
- **إصدار Native من claude code**: يكتشف تلقائيًا ملف `claude` الثنائي، ويهيئ وكيلًا شفافًا محليًا، ويضبط Zsh Shell Hook لإعادة توجيه حركة المرور تلقائيًا.
- يُوصى في هذا المشروع باستخدام Claude Code المثبت عبر npm.

إلغاء تثبيت وضع السجل:
```bash
ccv --uninstall
```

### استكشاف الأخطاء وإصلاحها (Troubleshooting)

إذا واجهت مشكلة في التشغيل، هناك حل نهائي لاستكشاف الأخطاء:
الخطوة الأولى: افتح Claude Code في أي مجلد؛
الخطوة الثانية: أعطِ Claude Code التعليمات التالية:
```
لقد قمت بتثبيت حزمة npm المسماة cc-viewer، لكن بعد تنفيذ ccv لا يعمل بشكل صحيح. افحص cli.js و findcc.js الخاصين بـ cc-viewer، وقم بالتكيف مع طريقة نشر Claude Code المحلية وفقًا للبيئة المحددة. حاول حصر نطاق التعديلات في findcc.js.
```
السماح لـ Claude Code بفحص الأخطاء بنفسه أكثر فعالية من استشارة أي شخص أو قراءة أي وثائق!

بعد اكتمال التعليمات أعلاه، سيتم تحديث findcc.js. إذا كان مشروعك يحتاج بشكل متكرر إلى النشر المحلي، أو إذا كان الكود المتفرع يحتاج إلى حل مشاكل التثبيت بشكل متكرر، فاحتفظ بهذا الملف. في المرة القادمة يمكنك نسخ الملف مباشرة. في المرحلة الحالية، العديد من المشاريع والشركات لا تنشر Claude Code على Mac، بل تستضيفه على الخادم، لذلك فصل المؤلف ملف findcc.js لتسهيل متابعة تحديثات الكود المصدري لـ cc-viewer.

### أوامر مساعدة أخرى

للاستعلام
```bash
ccv -h
```

### تجاوز الإعدادات (Configuration Override)

إذا كنت بحاجة إلى استخدام نقطة نهاية API مخصصة (مثل وكيل المؤسسة)، فقط قم بالتهيئة في `~/.claude/settings.json` أو عيّن متغير البيئة `ANTHROPIC_BASE_URL`. سيتعرف `ccv` عليه تلقائيًا ويعيد توجيه الطلبات بشكل صحيح.

### الوضع الصامت (Silent Mode)

بشكل افتراضي، يعمل `ccv` في الوضع الصامت عند تغليف `claude`، مما يضمن بقاء مخرجات الطرفية نظيفة ومتسقة مع التجربة الأصلية. يتم التقاط جميع السجلات في الخلفية ويمكن عرضها عبر `http://localhost:7008`.

بعد اكتمال التهيئة، استخدم أمر `claude` كالمعتاد. قم بزيارة `http://localhost:7008` لعرض واجهة المراقبة.

### التحديث التلقائي

يتحقق CC-Viewer تلقائيًا من التحديثات عند بدء التشغيل (مرة واحدة كل 4 ساعات كحد أقصى). ضمن نفس الإصدار الرئيسي (مثل 1.x.x → 1.y.z) يتم التحديث تلقائيًا ويسري عند إعادة التشغيل التالية. التغييرات عبر الإصدارات الرئيسية تعرض إشعارًا فقط.

يتبع التحديث التلقائي إعدادات Claude Code العامة في `~/.claude/settings.json`. إذا كان Claude Code قد عطّل التحديثات التلقائية (`autoUpdates: false`)، فسيتخطى CC-Viewer التحديث التلقائي أيضًا.
