# CC-Viewer

Система моніторингу запитів Claude Code, що в реальному часі перехоплює та візуалізує всі API-запити й відповіді Claude Code (оригінальний текст, без купюр). Допомагає розробникам стежити за своїм контекстом, щоб переглядати та діагностувати проблеми під час Vibe Coding.
Остання версія CC-Viewer також пропонує рішення для веб-програмування на сервері та інструменти для мобільного програмування. Запрошуємо застосовувати у своїх проектах — у майбутньому буде відкрито більше функцій плагінів із підтримкою хмарного розгортання.

## Використання

### Встановлення

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### Режим програмування

`ccv` є прямою заміною `claude` — усі аргументи передаються до `claude`, а Web Viewer запускається автоматично.

```bash
ccv                    # == claude（інтерактивний режим）
ccv -c                 # == claude --continue（продовжити попередню розмову）
ccv -r                 # == claude --resume（відновити розмову）
ccv -p "hello"         # == claude --print "hello"（режим друку）
ccv --d                # == claude --dangerously-skip-permissions（скорочення）
ccv --model opus       # == claude --model opus
```

Після запуску режиму програмування автоматично відкривається веб-сторінка.

На веб-сторінці можна безпосередньо використовувати claude, переглядати повні тіла запитів і зміни коду.

А ще — виглядає дуже круто — ви навіть можете програмувати з мобільного пристрою!

### Режим логера

⚠️ Якщо ви звикли використовувати нативні інструменти claude або плагін VS Code — скористайтеся цим режимом.

У цьому режимі запустіть ```claude``` або ```claude --dangerously-skip-permissions```.

Автоматично запускається процес логування, який записує журнали запитів до `~/.claude/cc-viewer/*yourproject*/date.jsonl`.

Запуск режиму логера:
```bash
ccv -logger
```

Якщо конкретний порт не відображається в консолі, перший порт за замовчуванням — `127.0.0.1:7008`. За наявності кількох екземплярів номери портів збільшуються: 7009, 7010 тощо.

Ця команда автоматично визначає спосіб встановлення Claude Code (NPM або Native Install) і адаптується відповідно.

- **NPM-версія claude code**: автоматично вставляє перехоплювальний скрипт у `cli.js` Claude Code.
- **Native-версія claude code**: автоматично виявляє бінарний файл `claude`, налаштовує локальний прозорий проксі та встановлює Zsh Shell Hook для автоматичного перенаправлення трафіку.
- Цей проект рекомендує встановлювати claude code через npm.

Видалення режиму логера:
```bash
ccv --uninstall
```

### Усунення несправностей (Troubleshooting)

Якщо виникають проблеми із запуском, є універсальний спосіб діагностики:
Крок 1: відкрийте claude code у будь-якій директорії;
Крок 2: дайте claude code таку інструкцію:
```
Я встановив npm-пакет cc-viewer, але після виконання ccv він не працює належним чином. Перегляньте cli.js та findcc.js пакету cc-viewer і адаптуйте їх до локального способу розгортання claude code відповідно до конкретного середовища. Намагайтеся обмежити зміни файлом findcc.js.
```
Дозволити Claude Code самостійно перевірити помилки — ефективніше, ніж консультуватися з будь-ким або читати будь-яку документацію!

Після виконання цієї інструкції буде оновлено `findcc.js`. Якщо ваш проект часто потребує локального розгортання або форкнутий код регулярно стикається з проблемами встановлення — збережіть цей файл і просто копіюйте його наступного разу. Оскільки зараз багато проектів і компаній розгортають claude code не на Mac, а на серверах, автор виніс `findcc.js` в окремий файл для зручного відстеження оновлень вихідного коду cc-viewer.

### Інші допоміжні команди

Довідка:
```bash
ccv -h
```

### Перевизначення конфігурації (Configuration Override)

Якщо вам потрібно використовувати власний API-ендпоінт (наприклад, корпоративний проксі), просто налаштуйте його в `~/.claude/settings.json` або встановіть змінну середовища `ANTHROPIC_BASE_URL`. `ccv` автоматично розпізнає та правильно перенаправить запити.

### Тихий режим (Silent Mode)

За замовчуванням `ccv` працює в тихому режимі під час обгортання `claude`, щоб вивід у терміналі залишався чистим і відповідав нативному досвіду. Усі журнали перехоплюються у фоновому режимі та доступні через `http://localhost:7008`.

Після налаштування використовуйте команду `claude` як зазвичай. Відкрийте `http://localhost:7008` для перегляду інтерфейсу моніторингу.

### Режим логера (перегляд повної сесії claude code)

<img width="1500" height="720" alt="image" src="https://github.com/user-attachments/assets/519dd496-68bd-4e76-84d7-2a3d14ae3f61" />

- Перехоплює в реальному часі всі API-запити від Claude Code — оригінальний текст, без купюр (це дуже важливо!!!)
- Автоматично розпізнає та позначає запити Main Agent і Sub Agent (підтипи: Plan, Search, Bash)
- Запити MainAgent підтримують Body Diff JSON — згорнуте відображення відмінностей від попереднього запиту MainAgent (показуються лише змінені/нові поля)
- Вбудована статистика використання токенів для кожного запиту (вхідні/вихідні токени, створення/читання кешу, відсоток влучань)
- Сумісний з Claude Code Router (CCR) та іншими проксі-сценаріями — резервне зіставлення запитів за шаблоном API-шляху

### Режим розмови

Натисніть кнопку «Режим розмови» у верхньому правому куті, щоб розібрати повну історію розмови Main Agent у вигляді чату:

<img width="1500" height="730" alt="image" src="https://github.com/user-attachments/assets/c973f142-748b-403f-b2b7-31a5d81e33e6" />

- Відображення Agent Team поки не підтримується
- Повідомлення користувача вирівняні праворуч (сині бульбашки), відповіді Main Agent — ліворуч (темні бульбашки)
- Блоки `thinking` згорнуті за замовчуванням, відображаються у Markdown; натисніть, щоб розгорнути та переглянути процес мислення; підтримується переклад одним кліком (функція ще нестабільна)
- Повідомлення з вибором користувача (AskUserQuestion) відображаються у форматі запитання-відповідь
- Двостороння синхронізація режимів: перехід до режиму розмови автоматично переходить до розмови, що відповідає вибраному запиту; повернення до режиму оригіналу автоматично переходить до вибраного запиту
- Панель налаштувань: можна перемикати стан згортання за замовчуванням для результатів інструментів і блоків мислення
- Перегляд розмови на телефоні: у режимі CLI на телефоні натисніть кнопку «Перегляд розмови» на верхній панелі, щоб відкрити панель перегляду розмови лише для читання

### Автоматичне оновлення

CC-Viewer автоматично перевіряє наявність оновлень під час запуску (не частіше одного разу на 4 години). У межах однієї мажорної версії (наприклад, 1.x.x → 1.y.z) оновлення відбувається автоматично й набирає чинності при наступному запуску. Для міжмажорних версій відображається лише сповіщення.

Автоматичне оновлення слідує глобальній конфігурації Claude Code `~/.claude/settings.json`. Якщо в Claude Code вимкнено автоматичне оновлення (`autoUpdates: false`), CC-Viewer також пропустить автоматичне оновлення.
