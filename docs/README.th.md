# CC-Viewer

ระบบตรวจสอบคำขอของ Claude Code ที่จับและแสดงผลคำขอและการตอบกลับ API ทั้งหมดของ Claude Code แบบเรียลไทม์ (ข้อความต้นฉบับ ไม่มีการตัดทอน) ช่วยให้นักพัฒนาตรวจสอบ Context ของตนเอง เพื่อทบทวนและแก้ไขปัญหาระหว่างกระบวนการ Vibe Coding
CC-Viewer เวอร์ชันล่าสุดยังมีโซลูชันการเขียนโปรแกรมผ่านเว็บแบบติดตั้งบนเซิร์ฟเวอร์ รวมถึงเครื่องมือสำหรับเขียนโปรแกรมบนมือถือ ยินดีต้อนรับทุกท่านนำไปใช้ในโปรเจกต์ของตนเอง ในอนาคตจะเปิดให้ใช้งานฟีเจอร์เพิ่มเติมและรองรับการติดตั้งบนคลาวด์

## วิธีใช้งาน

### การติดตั้ง

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### โหมดเขียนโปรแกรม

ccv เป็นตัวแทนโดยตรงของ claude ส่งผ่านพารามิเตอร์ทั้งหมดไปยัง claude พร้อมเปิด Web Viewer ด้วยกัน

```bash
ccv                    # == claude (โหมดโต้ตอบ)
ccv -c                 # == claude --continue (ต่อบทสนทนาครั้งก่อน)
ccv -r                 # == claude --resume (กู้คืนบทสนทนา)
ccv -p "hello"         # == claude --print "hello" (โหมดพิมพ์)
ccv --d                # == claude --dangerously-skip-permissions (ทางลัด)
ccv --model opus       # == claude --model opus
```

หลังจากเริ่มโหมดเขียนโปรแกรม หน้าเว็บจะเปิดขึ้นโดยอัตโนมัติ

คุณสามารถใช้ claude ได้โดยตรงบนหน้าเว็บ พร้อมทั้งดูข้อมูลคำขอแบบเต็มและดูการเปลี่ยนแปลงของโค้ด

และที่ดูเจ๋งกว่านั้นคือ คุณสามารถเขียนโปรแกรมบนมือถือได้เลย!


### โหมดล็อก

⚠️ หากคุณยังคงใช้เครื่องมือดั้งเดิมของ claude หรือปลั๊กอิน VS Code กรุณาใช้โหมดนี้

ในโหมดนี้ เมื่อเปิด ```claude``` หรือ ```claude --dangerously-skip-permissions```

ระบบจะเริ่มกระบวนการบันทึกล็อกโดยอัตโนมัติเพื่อบันทึกข้อมูลคำขอไปยัง ~/.claude/cc-viewer/*yourproject*/date.jsonl

เริ่มโหมดล็อก:
```bash
ccv -logger
```

เมื่อคอนโซลไม่สามารถแสดงพอร์ตที่เจาะจงได้ พอร์ตเริ่มต้นแรกคือ 127.0.0.1:7008 หากมีหลายอินสแตนซ์พร้อมกันจะเลื่อนต่อไปเป็น 7009, 7010

คำสั่งนี้จะตรวจจับวิธีการติดตั้ง Claude Code ในเครื่อง (NPM หรือ Native Install) โดยอัตโนมัติและปรับให้เข้ากัน

- **ติดตั้งผ่าน NPM**: ฉีดสคริปต์ดักจับเข้าไปใน `cli.js` ของ Claude Code โดยอัตโนมัติ
- **Native Install**: ตรวจจับไฟล์ไบนารี `claude` โดยอัตโนมัติ กำหนดค่า Transparent Proxy ในเครื่อง และตั้งค่า Zsh Shell Hook เพื่อส่งต่อทราฟฟิกโดยอัตโนมัติ
- โปรเจกต์นี้แนะนำให้ใช้ Claude Code ที่ติดตั้งผ่าน npm

ถอนการติดตั้งโหมดล็อก:
```bash
ccv --uninstall
```

### การแก้ไขปัญหา (Troubleshooting)

หากคุณพบปัญหาไม่สามารถเริ่มต้นได้ มีวิธีแก้ไขปัญหาขั้นสุดท้าย:
ขั้นตอนที่ 1: เปิด Claude Code ในไดเรกทอรีใดก็ได้
ขั้นตอนที่ 2: ให้คำสั่งแก่ Claude Code ดังนี้:
```
我已经安装了cc-viewer这个npm包，但是执行ccv以后仍然无法有效运行。查看cc-viewer的cli.js 和 findcc.js，根据具体的环境，适配本地的claude code的部署方式。适配的时候修改范围尽量约束在findcc.js中。
```
การให้ Claude Code ตรวจสอบข้อผิดพลาดด้วยตัวเองนั้นมีประสิทธิภาพมากกว่าการปรึกษาใครหรืออ่านเอกสารใดๆ!

หลังจากคำสั่งข้างต้นเสร็จสิ้น findcc.js จะถูกอัปเดต หากโปรเจกต์ของคุณต้องการติดตั้งในเครื่องบ่อยๆ หรือโค้ดที่ fork ไปต้องแก้ปัญหาการติดตั้งบ่อยๆ ให้เก็บไฟล์นี้ไว้ ครั้งต่อไปก็แค่คัดลอกไฟล์ ��นปัจจุบันหลายโปรเจกต์และบริษัทใช้ Claude Code ไม่ได้ติดตั้งบน Mac แต่เป็นการติดตั้งบนเซิร์ฟเวอร์ ดังนั้นผู้พัฒนาจึงแยกไฟล์ findcc.js ออกมา เพื่อให้ง่ายต่อการติดตามการอัปเดตซอร์สโค้ดของ cc-viewer

### คำสั่งเสริมอื่นๆ

ดูความช่วยเหลือ
```bash
ccv -h
```

### การกำหนดค่าแบบกำหนดเอง (Configuration Override)

หากคุณต้องการใช้ API endpoint แบบกำหนดเอง (เช่น พร็อกซีขององค์กร) เพียงกำหนดค่าใน `~/.claude/settings.json` หรือตั้งค่าตัวแปรสภาพแวดล้อม `ANTHROPIC_BASE_URL` `ccv` จะตรวจจับและส่งต่อคำขอได้อย่างถูกต้องโดยอัตโนมัติ

### โหมดเงียบ (Silent Mode)

���ามค่าเริ่มต้น `ccv` จะทำงานในโหมดเงียบเมื่อครอบ `claude` เพื่อให้แน่ใจว่าเอาต์พุตในเทอร์มินัลของคุณสะอาดเรียบร้อย สอดคล้องกับประสบการณ์ใช้งานแบบดั้งเดิม ล็อกทั้งหมดจะถูกจับในเบื้องหลังและสามารถดูได้ที่ `http://localhost:7008`

หลังจากกำหนดค่าเสร็จแล้ว ใช้คำสั่ง `claude` ตามปกติ เข้าถึง `http://localhost:7008` เพื่อดูหน้าจอตรวจสอบ

### โหมดสนทนา

คลิกปุ่ม "โหมดสนทนา" ที่มุมขวาบน เพื่อแปลงประวัติการสนทนาทั้งหมดของ Main Agent เป็นหน้าจอแชท:

<img width="1500" height="730" alt="image" src="https://github.com/user-attachments/assets/c973f142-748b-403f-b2b7-31a5d81e33e6" />

- ยังไม่รองรับการแสดงผล Agent Team
- ข้อความผู้ใช้จัดชิดขวา (บับเบิลสีน้ำเงิน), การตอบกลับของ Main Agent จัดชิดซ้าย (บับเบิลสีเข้ม)
- บล็อก `thinking` พับเก็บตามค่าเริ่มต้น แสดงผลเป็น Markdown คลิกเพื่อขยายดูกระบวนการคิด รองรับการแปลด้วยคลิกเดียว (ฟีเจอร์ยังไม่เสถียร)
- ข้อความเลือกของผู้ใช้ (AskUserQuestion) แสดงในรูปแบบถาม-ตอบ
- ซิงค์โหมดสองทาง: เมื่อสลับไปโหมดสนทนาจะเลื่อนไปยังการสนทนาที่ตรงกับคำขอที่เลือกโดยอัตโนมัติ เมื่อสลับกลับโหมดข้อความต้นฉบับจะเลื่อนไปยังคำขอที่เลือกโดยอัตโนมัติ
- แผงการตั้งค่า: สามารถสลับสถานะพับเก็บเริ่มต้นของผลลัพธ์เครื่องมือและบล็อกความคิด
- การเรียกดูสนทนาบนมือถือ: ในโหมด CLI บนมือถือ คลิกปุ่ม "เรียกดูสนทนา" ที่แถบด้านบน เพื่อเลื่อนเปิดมุมมองสนทนาแบบอ่านอย่างเดียว สำหรับเรียกดูประวัติการสนทนาทั้งหมดบนมือถือ

### การจัดการล็อก

ผ่านเมนูดรอปดาวน์ CC-Viewer ที่มุมซ้ายบน:

<img width="1200" height="672" alt="image" src="https://github.com/user-attachments/assets/8cf24f5b-9450-4790-b781-0cd074cd3b39" />

- นำเข้าล็อกในเครื่อง: เรียกดูไฟล์ล็อกย้อนหลัง จัดกลุ่มตามโปรเจกต์ เปิดในหน้าต่างใหม่
- โหลดไฟล์ JSONL ในเครื่อง: เลือกไฟล์ `.jsonl` ในเครื่องโดยตรงเพื่อดู (รองรับสูงสุด 500MB)
- บันทึกล็อกปัจจุบันเป็น: ดาวน์โหลดไฟล์ล็อก JSONL ที่กำลังตรวจสอบอยู่
- รวมล็อก: รวมไฟล์ล็อก JSONL หลายไฟล์เป็นเซสชันเดียวเพื่อการวิเคราะห์แบบรวม
- ดู Prompt ผู้ใช้: ดึงและแสดงข้อมูลนำเข้าของผู้ใช้ทั้งหมด รองรับสามโหมดการดู — โหมดต้นฉบับ (เนื้อหาดิบ), โหมดบริบท (แท็กระบบยุบได้), โหมดข้อความ (ข้อความธรรมดา); คำสั่ง slash (`/model`, `/context` ฯลฯ) แสดงเป็นรายการอิสระ; แท็กที่เกี่ยวข้องกับคำสั่งจะถูกซ่อนจากเนื้อหา Prompt โดยอัตโนมัติ
- ส่งออก Prompt เป็น TXT: ส่งออก Prompt ผู้ใช้ (ข้อความธรรมดา ไม่รวมแท็กระบบ) เป็นไฟล์ `.txt` ในเครื่อง

### อัปเดตอัตโนมัติ

CC-Viewer ตรวจสอบการอัปเดตโดยอัตโนมัติเมื่อเริ่มต้น (สูงสุดทุก 4 ชั่วโมง) ภายในเวอร์ชันหลักเดียวกัน (เช่น 1.x.x → 1.y.z) จะอัปเดตโดยอัตโนมัติและมีผลเมื่อเริ่มต้นครั้งถัดไป การเปลี่ยนเวอร์ชันหลักจะแสดงเฉพาะการแจ้งเตือน

การอัปเดตอัตโนมัติเป็นไปตามการตั้งค่าส่วนกลางของ Claude Code ใน `~/.claude/settings.json` หาก Claude Code ปิดการอัปเดตอัตโนมัติ (`autoUpdates: false`) CC-Viewer จะข้ามการอัปเดตอัตโนมัติเช่นกัน
